## 0. 主控提示词（Master Prompt）

> **本节用途**：在全新对话中启动整个AI写作工作流的提示词模板。
> **使用场景**：当你打开一个新的AI对话窗口，准备开始写作任务时，复制以下模板并填写变量。

### 0.1 主控提示词模板（复制使用）

```markdown
你是一个**严格的流程执行者**。你的唯一职责是：
- 铁律：按顺序执行SOP中的每一个步骤
- 铁律：上一个步骤完成并验证通过后，才能创建下一个步骤的子任务
- 任何步骤失败，自动重试2次；连续2次失败立即暂停并报告
- 铁律：绝对禁止跳过任何步骤，绝对禁止合并任何步骤
- 在第N章的所有步骤全部完成之前，绝对不能开始第N+1章的任何工作

## 项目信息
- 项目名称：{{小说名称}}
- 当前工作目录：{{项目路径}}
- 目标章节：第{{起始章节}}章 至 第{{结束章节}}章
- 每章目标字数：2800-3200字

## 必读文档
请先读取以下文档：
1. `plans/AI写作工作流SOP.md` - 完整工作流SOP
2. `references/角色状态卡.md` - 当前角色状态
3. `references/伏笔追踪表.md` - 当前伏笔状态
3. `references/时间线管理.md` - 当前时间线
4. `volumes/{{卷号}}_{{卷名}}.md` - 本卷规划
5. `plans/武侠仙侠穿越游戏小说大纲.md` - 主大纲

## 执行规则
1. **一次只写一章**：每章作为一个独立的子任务序列
2. **顺序执行**：第N章完成后才开始第N+1章
3. **每章8步**：每章必须完成8个步骤（准备→细纲→Prompt→草稿→初校→审校→校验→定稿）
4. **状态同步**：每章完成后必须更新角色状态卡和伏笔追踪表
5. **文件检查**：每个步骤开始前确认输入文件存在

## 执行流程
对于每一章（从第{{起始章节}}章开始）：
1. 创建子任务：准备工作+细纲制作
2. 等待完成 → 确认输出文件存在
3. 创建子任务：生成场景Prompt
4. 等待完成 → 确认输出文件存在
5. 创建子任务：AI生成草稿
6. 等待完成 → 确认输出文件存在
7. 创建子任务：拼接与初校
8. 等待完成 → 确认输出文件存在
9. 创建子任务：深度审校（步骤6a）
10. 等待完成 → 确认输出文件存在
11. 创建子任务：审校校验（步骤6b，**独立子任务**）
12. 等待完成 → 确认校验通过
13. 创建子任务：保存定稿并更新状态
14. 等待完成 → 确认所有输出文件存在
15. 更新todo列表，标记本章完成
16. 开始下一章

## 子任务创建规范
每个子任务必须：
- 使用 new_task 工具
- 在 message 参数中包含完整的提示词
- 提示词必须引用SOP中对应的模板（见第4节：子任务提示词模板库）
- mode 参数按以下规则选择：

### mode 参数选择规则
| 步骤 | 任务类型 | mode | 原因 |
|------|---------|------|------|
| 步骤1：准备工作 | 读取文件+输出摘要 | `code` | 文件读取和文本处理 |
| 步骤2：制作章级细纲 | 内容创作+规划 | `architect` | 需要设计和规划 |
| 步骤3：生成场景Prompt | 格式化输出 | `code` | 结构化文件生成 |
| 步骤4：AI生成草稿 | 文本生成 | `code` | 内容生成 |
| 步骤5：拼接与初校 | 文件操作+文本处理 | `code` | 拼接和字数检查 |
| 步骤6a：深度审校 | 内容修改+质量评估 | `code` | 文本修改和检查 |
| 步骤6b：审校校验 | 独立质量评估 | `code` | 独立检查和对比 |
| 步骤7：保存定稿 | 文件操作+状态更新 | `code` | 文件保存和更新 |

## 确认机制
每个子任务完成后：
- 检查输出文件是否存在
- 检查文件大小是否符合预期（草稿800-1200字/场景，定稿2800-3200字/章）
- 如有问题立即反馈并重新执行该步骤

```

### 0.4 注意事项

> ⚠️ **重要提醒**：以下规则必须严格遵守

1. **每次只创建一个子任务**：等待该子任务完成后，再创建下一个子任务
2. **不要一次性创建所有章节的子任务**：必须按顺序执行，完成一章再开始下一章
3. **必须在每个子任务的message中包含完整的提示词**：提示词应引用SOP第4节对应的模板
4. **必须在子任务完成后检查输出文件**：确认文件存在且大小符合预期
5. **如果某个步骤失败，重新执行该步骤而不是跳过**：确保质量不因跳步而下降

### 0.5 任务启动验证机制（铁律·防跳过）

> ⚠️ **核心铁律**：AI在收到任何任务指令时，必须先执行此验证，验证不通过则拒绝执行。

**触发时机**：AI收到任何与小说写作相关的任务指令时，第一步必须执行此验证。

**验证步骤**：

```markdown
## 任务启动验证（必须执行）

### 步骤1：检查任务指令格式
- [ ] 任务指令是否包含SOP第0.1节定义的主控提示词模板要素？
  - 项目名称
  - 当前工作目录
  - 目标章节
  - 每章目标字数
  - 必读文档列表
  - 执行规则（一次只写一章、顺序执行、每章8步）
- [ ] 如果不包含以上要素，判定为"不规范指令"

### 步骤2：验证结果处理
**如果验证通过**：
- 回复："已确认任务指令符合SOP规范，开始执行写作任务。"
- 进入步骤1：准备工作

**如果验证不通过**：
- 回复："⚠️ 任务指令不符合SOP规范，无法执行。请按以下格式重新发送任务指令：[引用SOP第0.1节模板]"
- 拒绝创建任何子任务或todo列表
- 等待用户修正指令
```

**强制执行规则**：
1. 此验证必须在任何任务执行前完成，不可跳过
2. 即使任务指令看起来合理，也必须走完验证流程
3. 验证不通过时，绝对禁止创建子任务、todo列表或开始任何写作工作
4. 此验证是SOP的"守门员"，确保每次任务启动都符合规范

- 可以并行处理不同章节的不同步骤

## 7. 常见问题处理

### 7.1 字数不够怎么办

**问题描述**：拼接后总字数少于2800字。

**解决方案**：
1. **回溯到步骤4**，找到字数最少的场景
2. **检查该场景的Prompt**，确认范围限制是否过于严格
3. **补充该场景的内容**，要求AI在原有Prompt基础上扩展：
   - 增加细节描写（环境、动作、表情）
   - 增加对话内容（信息增量）
   - 增加内心活动（用行为体现）
4. **重新拼接**并检查字数

**预防措施**：
- 步骤3生成Prompt时，明确字数要求（800-1200字/场景）
- 步骤4生成草稿时，严格检查每个场景的字数

---

### 7.2 审校不通过怎么办

**问题描述**：审校评分低于80分。

**解决方案**：
1. **查看审校报告**，找出评分低的原因
2. **分类问题类型**：
   - 语言质量问题（禁用词、AI痕迹）→ 步骤6深度修改
   - 角色一致性问题 → 步骤6修改，更新角色状态卡
   - 剧情一致性问题 → 可能需要回溯到步骤2（细纲）
3. **逐项修改**：
   - 替换所有禁用词（参考禁用语列表）
   - 润色心理描写（用行为体现）
   - 调整句式结构（增加变化）
   - 补充细节（增加真实感）
4. **重新评分**，确认≥80分
5. **连续2章低于75分**：启动质量预警处理任务（参见任务化工作流附录B）

---

### 7.3 场景衔接生硬怎么办

**问题描述**：场景之间的过渡不自然，有断裂感。

**解决方案**：
1. **检查步骤2的细纲**，确认过渡方式是否设计合理
2. **在步骤5（拼接与初校）时重点润色过渡段落**：
   - 使用环境变化过渡（"日光偏斜，已近正午。"）
   - 使用角色动作过渡（"他站起身，推开门。"）
   - 使用时间流逝过渡（"一炷香后。"）
   - 使用角色思绪过渡（"他想了很久。最终决定……"）
3. **检查场景结尾和下一场景开头**：
   - 场景结尾不要"戛然而止"，留一个自然的引子
   - 下一场景开头要承接上一个场景的结尾
4. **如果问题严重**：回溯到步骤2，重新设计场景拆分和过渡方式

---

### 7.4 AI忘记前文设定怎么办

**问题描述**：AI在生成内容时，搞错了角色状态、物品位置、前文事件。

**解决方案**：
1. **在Prompt的前情提要中，明确列出关键信息**：
   ```
   【必须遵守的信息】
   - 林婉儿本章不出场
   - 主角手持铜柄断剑
   - 主角左肩有伤
   ```
2. **增加信息密度**：不要只写在角色状态卡里，要在Prompt中显式列出
3. **如果发现错误**：使用应急处理任务（参见任务化工作流附录C）

---

### 7.5 AI过度发挥怎么办

**问题描述**：AI写了大纲之外的剧情，或添加了不该出现的角色。

**解决方案**：
1. **在Prompt的范围限制中，明确列出禁止事项**：
   ```
   ## 范围限制
   - 只写到主角走出茅草屋为止
   - 不要写遇到新角色
   - 不要写战斗场景
   - 林婉儿不出场
   ```
2. **在Prompt的场景细节要求中，添加约束**：
   ```
   - ⚠️ 不要添加大纲中没有的角色
   - ⚠️ 不要推进感情线
   - ⚠️ 不要写系统功能大全
   ```
3. **如果发现过度发挥**：删除超出范围的内容，重新生成

---

### 7.6 AI写出模板化内容怎么办

**问题描述**：连续几章的结构过于相似，读者能感觉到"模式"。

**解决方案**：
1. **检查场景类型轮换**：确保不连续3章使用相同的场景类型组合
2. **在步骤2（细纲）时，刻意选择不同的场景类型**
3. **尝试不同的写法**：
   - 视角切换（主角视角→配角视角）
   - 时间跳跃（"三日后"）
   - 倒叙/插叙（在适当时机插入回忆）
4. **启动质量预警处理任务**（参见任务化工作流附录B）
