## 二、流水线工作流程

### 2.1 每日写作流程（约3-4小时）

```
┌─────────────────────────────────────────────────────────────┐
│ 步骤1：准备工作（15分钟）                                     │
├─────────────────────────────────────────────────────────────┤
│ 1. 打开【references/角色状态卡.md】，确认当前角色状态           │
│ 2. 打开【references/伏笔追踪表.md】，确认伏笔状态               │
│ 3. 打开【logs/剧情进度表.md】，确认当前进度                     │
│ 4. 回顾前3章定稿内容（快速浏览）                               │
└─────────────────────────┬───────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│ 步骤2：制作章级细纲（20分钟）                                  │
├─────────────────────────────────────────────────────────────┤
│ 1. 复制【templates/章级细纲模板.md】到对应目录                │
│    → outlines/volXX/chXXX_细纲.md                           │
│ 2. 填入本章核心事件                                           │
│ 3. 拆分2-4个场景，明确每个场景的核心动作                       │
│ 4. 设定结尾钩子                                              │
│ 5. 填写细节要求（前文物品、伏笔呼应、角色状态）                 │
└─────────────────────────┬───────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│ 步骤3：生成Prompt（15分钟/场景）                               │
├─────────────────────────────────────────────────────────────┤
│ 1. 复制对应场景模板到prompts目录                              │
│    → prompts/volXX/chXXX_场景X.md                           │
│ 2. 填入细纲内容                                              │
│ 3. 附带角色状态卡（复制references中的相关内容）                │
│ 4. 附带禁用语列表（复制references/禁用语列表.md内容）          │
│ 5. 添加写作风格要求                                           │
└─────────────────────────┬───────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│ 步骤4：AI生成（10分钟/场景）                                   │
├─────────────────────────────────────────────────────────────┤
│ 1. 将Prompt内容发送给AI                                       │
│ 2. 等待AI生成（约800-1200字/场景）                            │
│ 3. 将生成内容保存到【drafts/volXX/chXXX_场景X_草稿.md】       │
│ 4. 快速检查是否符合场景要求                                   │
│ 5. 如不满意，调整Prompt重新生成                              │
└─────────────────────────┬───────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│ 步骤5：拼接与初校（20分钟）                                    │
├─────────────────────────────────────────────────────────────┤
│ 1. 将2-4个场景草稿拼接成一章                                  │
│ 2. 润色场景过渡段落，确保衔接自然                             │
│ 3. 检查字数是否在2800-3200字范围内                            │
│ 4. 初步检查是否有明显AI痕迹                                   │
└─────────────────────────┬───────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│ 步骤6：深度审校（30-40分钟）                                   │
├─────────────────────────────────────────────────────────────┤
│ 1. 复制【一致性检查清单】到【reviews/chXXX_检查清单.md】      │
│ 2. 逐项检查                                                    │
│   · 角色一致性                                                  │
│   · 剧情一致性                                                  │
│   · 力量体系一致性                                              │
│   · 语言质量                                                    │
│   · 节奏检查                                                    │
│ 3. 修改发现的问题                                              │
│ 4. 去除AI痕迹（替换禁用词汇、润色心理描写等）                   │
│ 5. 注入个人风格（调整句式、增加细节）                           │
└─────────────────────────┬───────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│ 步骤7：保存定稿（5分钟）                                       │
├─────────────────────────────────────────────────────────────┤
│ 1. 将定稿保存到【chapters/第XXX章_章名.md】                   │
│ 2. 更新【references/角色状态卡.md】                          │
│ 3. 更新【references/伏笔追踪表.md】                          │
│ 4. 更新【logs/剧情进度表.md】                                │
│ 5. 填写【logs/每日写作日志.md】                              │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 每周维护流程（约1小时，建议周末进行）

```
┌─────────────────────────────────────────────────────────────┐
│ 步骤1：本周回顾（15分钟）                                     │
├─────────────────────────────────────────────────────────────┤
│ 1. 统计本周完成章节数                                         │
│ 2. 对比卷级规划，检查进度是否偏差                             │
│ 3. 检查是否有偏离大纲的内容                                   │
│ 4. 填写【logs/每周维护记录.md】                              │
└─────────────────────────┬───────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│ 步骤2：深度一致性检查（20分钟）                                │
├─────────────────────────────────────────────────────────────┤
│ 1. 使用【深度一致性检查清单】                                │
│ 2. 检查角色成长是否符合预期                                   │
│ 3. 检查配角戏份是否合理                                       │
│ 4. 检查感情发展节奏是否正确                                   │
│ 5. 检测模板化倾向（连续3章是否有相似结构）                    │
│ 6. 检测节奏问题（是否连续5章没有高潮）                        │
└─────────────────────────┬───────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│ 步骤3：更新状态卡（10分钟）                                    │
├─────────────────────────────────────────────────────────────┤
│ 1. 更新所有角色的当前境界                                     │
│ 2. 更新所有角色的当前状态                                     │
│ 3. 更新所有角色的位置                                         │
│ 4. 更新关键物品状态                                           │
└─────────────────────────┬───────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│ 步骤4：更新伏笔追踪表（10分钟）                                │
├─────────────────────────────────────────────────────────────┤
│ 1. 登记本周新埋设的伏笔                                      │
│ 2. 标记本周回收的伏笔                                         │
│ 3. 规划下周伏笔回收计划                                       │
│ 4. 检查是否有遗漏需要回收的伏笔                               │
└─────────────────────────┬───────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│ 步骤5：规划下周内容（5分钟）                                   │
├─────────────────────────────────────────────────────────────┤
│ 1. 确认下周要写的章节范围                                     │
│ 2. 制作下周章节细纲（可批量制作3-5章细纲）                    │
│ 3. 确认关键节点                                               │
│ 4. 确认伏笔回收计划                                           │
└─────────────────────────────────────────────────────────────┘
```

## 三、第一次启动流程（初始化项目）

### 3.1 初始化步骤

```
步骤1：创建目录结构
→ 执行本文档中的"创建目录结构"命令
→ 所有目录创建完成

步骤2：复制参考资料
→ 从大纲中提取角色信息，填入【references/角色状态卡.md】
→ 从大纲中提取伏笔信息，填入【references/伏笔追踪表.md】
→ 从大纲中提取世界观设定，填入【references/世界观设定.md】
→ 从大纲中提取力量体系，填入【references/力量体系.md】
→ 从AI写作控制指南中提取禁用语列表，填入【references/禁用语列表.md】
→ 准备写作风格参考，填入【references/风格参考.md】

步骤3：复制模板文件
→ 从AI写作控制指南中提取各模板，分别保存到templates目录

步骤4：制作第一卷规划
→ 复制卷级规划模板到【volumes/第一卷_卷纲_穿越觉醒.md】
→ 根据大纲第一卷内容填写

步骤5：制作前3章细纲
→ 复制章级细纲模板到【outlines/vol01/ch001_细纲.md】等
→ 根据大纲填写前3章细纲

步骤6：试写第1章
→ 按照每日写作流程，完成第1章的完整写作流程
→ 检验工作流是否顺畅

步骤7：开始正式写作
→ 按照每日写作流程，逐章写作
```

## 六、质量管控

### 6.1 每章质量评分标准

| 维度 | 权重 | 说明 |
|------|------|------|
| 剧情推进 | 15% | 本章是否推进了剧情 |
| 角色塑造 | 15% | 角色行为是否符合性格 |
| 对话质量 | 15% | 对话是否自然、符合角色 |
| 战斗描写 | 10% | 战斗是否有策略性、有细节 |
| 情感表达 | 10% | 感情是否通过行为和细节体现 |
| 节奏控制 | 10% | 节奏是否符合预期 |
| 结尾钩子 | 10% | 结尾是否有吸引力 |
| 语言质量 | 10% | 是否有AI痕迹、是否流畅 |
| 一致性 | 5% | 是否与前文一致 |

### 6.2 质量预警机制

- **连续2章评分低于75分**：暂停写作，分析问题原因，调整Prompt策略
- **连续3章结构相似**：强制更换下一章的场景拆分方式
- **连续5章没有高潮**：检查卷级规划，确认是否需要调整节奏
- **伏笔超过50章未回收**：立即规划回收章节

## 七、应急处理

### 7.1 AI写崩了怎么办

| 问题 | 解决方案 |
|------|----------|
| AI忘记前文设定 | 在Prompt开头添加"【重要提醒】"区块，列出关键信息 |
| AI严重注水 | 在Prompt中明确字数要求，删除所有无意义描写 |
| AI过度发挥 | 在Prompt中添加"【严格限制】不要添加以下剧情：[列出]" |
| AI写出模板化内容 | 更换Prompt结构，使用不同的场景拆分方式 |
| AI写出完全不可用的内容 | 换用备用AI模型，或者自己写这一章 |

### 7.2 发现严重逻辑错误

1. **立即停止写作**，不要继续写后续章节
2. **定位问题**：从第几章开始出现逻辑错误
3. **回退修改**：修改问题章节的细纲和定稿
4. **更新状态卡**：修正角色状态卡中的错误信息
5. **更新Prompt**：修改后续Prompt中的相关信息
6. **重新审校**：检查已写的后续章节是否受到影响

## 八、工具推荐

### 8.1 推荐工具

| 用途 | 工具 | 说明 |
|------|------|------|
| 文档管理 | VS Code / Notion / 飞书 | 管理所有文档 |
| 表格管理 | Excel / 飞书表格 | 角色状态卡、伏笔追踪表 |
| AI模型 | 多个备用模型 | 主力模型+2-3个备选模型 |
| 字数统计 | VS Code插件 / Word | 统计每章字数 |
| 版本控制 | Git | 备份和版本管理（可选） |

### 8.2 AI模型选择建议

- **主力模型**：选择中文写作能力最强的模型
- **备用模型1**：用于战斗场景（选择擅长动作描写的模型）
- **备用模型2**：用于感情场景（选择擅长情感表达的模型）
- **备用模型3**：用于日常对话（选择对话自然的模型）
