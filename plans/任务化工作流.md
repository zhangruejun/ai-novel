# 《剑道通神》任务化写作工作流（无限画布版）

## 设计目标

解决AI在多章节写作时"第一步严格，后续跳步"的问题。将每章写作的7步流程拆分成**独立任务**，每个任务可以启动**新的对话实例**执行，确保每步都严格按标准完成。

---

## 任务1：准备工作

**任务指令**（复制到新对话）：

```
你现在要执行《剑道通神》第写作任务的"准备工作"步骤。

## 必读文档
1. 先读取 `写作执行手册.md`
2. 再读取 `plans/工作流.md`

## 任务内容
1. 读取 `references/角色状态卡.md`，记录当前所有角色的状态
2. 读取 `references/伏笔追踪表.md`，记录当前伏笔状态
3. 读取 `logs/每日写作日志.md`，确认当前进度
4. 列出 `chapters/` 目录下最近3章的文件名，并读取其内容摘要（每章前500字）

## 输出格式
请按以下格式输出准备工作摘要：

### 当前进度
- 已完成章节：第X章 - 第X章
- 下一章待写：第XXX章
- 当前字数统计：XXXX字

### 角色状态摘要
（列出所有活跃角色的关键状态信息）

### 伏笔状态摘要
（列出所有未回收伏笔及其紧急程度）

### 前3章内容摘要
（每章用3-5句话概括核心事件和结尾钩子）

### 确认清单
- [ ] 已读写作执行手册
- [ ] 已读工作流
- [ ] 已确认角色状态
- [ ] 已确认伏笔状态
- [ ] 已回顾前3章内容

确认以上信息无误后，任务完成。下一步将启动新对话执行"任务2：制作细纲"。
```

**输出验证标准**：
- [ ] 输出包含当前进度的准确描述
- [ ] 输出包含所有活跃角色的状态摘要
- [ ] 输出包含所有未回收伏笔的列表
- [ ] 输出包含前3章的内容摘要
- [ ] 确认清单全部勾选

**输出文件**：无文件输出，仅输出文本摘要供下一步使用

## 任务2：制作章级细纲

**任务指令**（复制到新对话）：

```
你现在要执行《剑道通神》写作任务的"制作章级细纲"步骤。

## 输入信息
（粘贴任务1的输出摘要）

## 必读文档
1. 读取 `templates/章级细纲模板.md`
2. 读取 `volumes/第一卷_卷纲_穿越觉醒.md`（或对应卷规划）
3. 读取 `plans/武侠仙侠穿越游戏小说大纲.md` 中对应章节规划

## 任务内容
1. 根据大纲和卷规划，确定第XXX章的核心事件
2. 拆分2-4个场景，明确每个场景的核心动作
3. 设定结尾钩子
4. 填写细节要求（前文物品、伏笔呼应、角色状态）

## 输出格式
完整填写章级细纲（参照章级细纲模板），保存到 `outlines/vol01/chXXX_细纲.md`

### 必须包含的内容
- 基本信息（章节序号、字数目标、场景数量、场景类型）
- 本章在卷中的位置（承上、启下、本章作用）
- 本章核心（核心事件、情绪基调、核心冲突）
- 前章衔接（上一章结尾、本章开头、连续性注意事项）
- 场景拆分（2-4个场景，每个场景必须有：场景类型、时间地点、出场角色、核心动作、对话要点、过渡方式）
- 结尾钩子（钩子类型、具体内容、预期效果）
- 细节要求（前文物品提及、伏笔处理、角色状态提醒、战斗信息等）

完成后，将细纲内容输出，并确认以下清单：

### 确认清单
- [ ] 场景数量在2-4个之间
- [ ] 每个场景都有明确的场景类型（A/B/C/D/E/F）
- [ ] 每个场景都有核心动作描述
- [ ] 每个场景都有对话要点
- [ ] 结尾有明确的钩子设计
- [ ] 与上一章衔接自然
- [ ] 与大纲规划一致

确认无误后，任务完成。下一步将启动新对话执行"任务3：生成Prompt"。
```

**输出验证标准**：
- [ ] 细纲文件已保存到正确路径
- [ ] 场景数量在2-4个之间
- [ ] 每个场景都有完整的6项信息
- [ ] 结尾钩子设计具体可执行
- [ ] 与大纲和卷规划一致

**输出文件**：`outlines/volXX/chXXX_细纲.md`

## 任务3：生成场景Prompt

**任务指令**（复制到新对话）：

```
你现在要执行《剑道通神》写作任务的"生成场景Prompt"步骤。

## 输入信息
- 读取 `outlines/volXX/chXXX_细纲.md`
- 读取 `references/角色状态卡.md`
- 读取 `references/禁用语列表.md`
- 读取 `templates/Prompt模板集合.md`
- 读取上一章定稿的最后200-300字（用于前情提要）

## 任务内容
为第XXX章的**每个场景**创建独立的Prompt文件。

每个Prompt必须包含：
1. 通用结构（角色设定 + 前情提要 + 当前状态摘要 + 必须遵守的信息）
2. 章节信息（章名、核心事件、情绪基调、字数要求）
3. 场景信息（地点、时间、出场角色、核心动作、对话要点）
4. 写作风格要求
5. 禁用语列表
6. 范围限制（只写到什么为止，不要写什么）

## 输出格式
为每个场景创建一个Prompt文件，保存到 `prompts/volXX/` 目录：
- `chXXX_场景1.md`
- `chXXX_场景2.md`
- `chXXX_场景3.md`
- （如有场景4）`chXXX_场景4.md`

每个文件内容按Prompt模板集合中的格式填写。

### 确认清单
- [ ] 每个场景都有独立的Prompt文件
- [ ] 每个Prompt都包含前情提要模块
- [ ] 每个Prompt都附带了角色状态卡相关内容
- [ ] 每个Prompt都附带了禁用语列表
- [ ] 每个Prompt都有明确的范围限制
- [ ] 文件命名符合规范（chXXX_场景X.md）

确认无误后，任务完成。下一步将启动新对话执行"任务4：AI生成草稿"。
```

**输出验证标准**：
- [ ] 每个场景都有独立的Prompt文件
- [ ] Prompt文件数量与细纲中的场景数量一致
- [ ] 每个Prompt都包含必需的所有模块
- [ ] 文件命名符合规范

**输出文件**：`prompts/volXX/chXXX_场景X.md`（每个场景一个文件）

## 任务4：AI生成草稿

**任务指令**（复制到新对话）：

```
你现在要执行《剑道通神》写作任务的"AI生成草稿"步骤。

## 输入信息
- 读取 `prompts/volXX/chXXX_场景1.md`
- 读取 `prompts/volXX/chXXX_场景2.md`
- 读取 `prompts/volXX/chXXX_场景3.md`
- （如有场景4）读取 `prompts/volXX/chXXX_场景4.md`

## 任务内容
按照每个场景的Prompt要求，依次生成草稿内容。

**重要规则**：
1. 每次只生成**一个场景**的内容
2. 生成完成后，等待确认再继续下一个场景
3. 每个场景控制在800-1200字之间
4. 严格按照Prompt中的范围限制执行，不要过度发挥

## 执行流程
请先生成**场景1**的草稿内容，输出后等待确认。
确认通过后，再生成场景2，依此类推。

## 输出格式
每个场景的输出格式：

### 场景X草稿
（生成的正文内容）

---
字数统计：XXX字
场景类型：[A/B/C/D/E/F]
核心动作完成度：[是/否]
结尾衔接：[是否自然过渡到下一场景]

### 场景1
（生成内容）

[等待确认后继续场景2...]
```

**输出验证标准**：
- [ ] 每个场景草稿字数在800-1200字之间
- [ ] 每个场景都完成了核心动作
- [ ] 场景之间的过渡自然
- [ ] 没有超出Prompt范围限制的内容
- [ ] 没有使用禁用语

**输出文件**：`drafts/volXX/chXXX_场景X_草稿.md`（每个场景一个文件）

## 使用指南

### 如何开始新章节写作

1. **确定要写的章节号**（如第006章）
2. **复制"任务1"的指令到新对话**，执行准备工作
3. **确认任务1输出符合验证标准**后，复制"任务2"指令到新对话
4. **依此类推**，直到任务7完成
5. **一章完成后**，再开始下一章的任务1

### 注意事项

- ⚠️ **每个任务必须使用新的对话实例**，不要在同一对话中连续执行多个任务
- ✅ 每个任务完成后，检查输出是否符合验证标准
- ✅ 如果某任务输出不符合标准，要求AI修改后再进入下一步
- ✅ 任务之间通过"输入信息"衔接，确保信息传递完整
- ✅ 可以并行处理不同章节的不同任务（如任务4生成第6章草稿时，任务2可以同时制作第7章细纲）

### 多章节并行工作流示例

```
对话A：第006章 任务1（准备）→ 完成
对话B：第006章 任务2（细纲）→ 完成
对话C：第006章 任务3（Prompt）→ 完成
对话D：第006章 任务4（生成）→ 进行中
对话E：第007章 任务1（准备）→ 可开始
对话F：第007章 任务2（细纲）→ 可开始
```
