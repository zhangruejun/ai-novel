# Prompt模板集合

> 使用说明：
> 1. 每次生成内容前，复制对应模板并填写
> 2. **每个Prompt必须包含前情提要模块**（见下方通用结构）
> 3. 分场景生成时，每个场景使用一个Prompt
> 4. 生成后保存至：prompts/volXX/chXXX_场景X.md
> 5. **前情提要信息来源**：
>    - 角色状态 → `references/角色状态卡.md`
>    - 时间线/摘要 → `references/时间线管理.md`
>    - 活跃伏笔 → `references/伏笔追踪表.md`
>    - 禁用语 → `references/禁用语列表.md`

## 写作风格要求
- 战斗描写：突出剑道特色，强调弱点分析+策略，有动作细节，避免"你来我往"
- 对话风格：
  - 李长风：话少有力，分析在前，行动果断
  - 林婉儿：温和带锋，善用问句，危机时冷静得出奇
  - 张云义：直来直去，激动时语速加快
- 环境描写：简洁有画面感，不要堆砌辞藻
- 节奏控制：[快节奏/慢节奏/张弛有度]
- 游戏系统感：系统面板只在关键时刻显示（突破、获得奖励），不要频繁弹出

## 禁用语（严格执行，完整列表见references/禁用语列表.md）
核心禁用：
- 然而、与此同时、值得注意的是、不得不提
- 他感到一阵复杂的情绪涌上心头/内心久久不能平静
- 嘴角勾起一抹微笑/冷笑
- 眼中闪过一丝精光/杀意
- 倒吸一口凉气
- 犹如/宛如/如同/仿佛（慎用，可以偶尔用一两次）
- 一道寒光闪过/剑气纵横/你来我往

## 范围限制
- 只写到[具体事件]为止
- 不要写[明确不要写什么]
- 不要添加大纲中没有的角色
- 不要推进[某条]感情线（如指定）
- **时间限制**：本章时间跨度不超过[X小时/半天/一天]
- **不要元叙事**：角色不能用"第X章""这一章"等概念
```

## 2. 战斗场景专用Prompt

> 适用于有战斗的章节。填写前删除此说明。

```
[通用结构]

## 战斗场景信息
- 对手信息：[对手名称][当前境界][武器][特点][弱点（必填！）]
- 战斗原因：[为什么打起来]
- 战斗地点：[具体环境，是否影响战斗，有无天然陷阱]
- 旁观者：[如有，列出]

## 战斗阶段设计
1. 试探阶段（约300字）：双方交手，互相试探，主角利用「游戏先知」分析对手弱点
2. 交锋阶段（约300字）：对手发动真正攻势，主角稍处劣势（不能直接碾压）
3. 分析阶段（约200字）：主角找到弱点或规律（必须明确说出分析过程）
4. 反击阶段（约300字）：主角利用弱点，结合地形，扭转局势
5. 决胜阶段（约200字）：主角施展关键一击

## 战斗特色
- 本场战斗特色：[如"剑意克制刀法缺口""利用落石陷阱""地形借力"，不要千篇一律]
- 断剑使用：[本场战斗断剑是否发挥特效，吸收了什么能量，发出什么颜色微光]
- 战斗代价：[主角是否受伤/消耗/获得感悟]
- 战斗结果：[胜/负/平/中断]

## 注意事项
- 战斗描写要有具体动作细节（剑从哪个角度，对方如何应对）
- 强调策略和智慧，不只是境界压制
- 战斗中可穿插内心分析，但用行动体现，不要大段内心独白
- 境界差超过半个大境界时，主角必须有特殊条件才能获胜（地形/陷阱/断剑/游戏先知）
```

## 3. 感情场景专用Prompt

> 适用于感情培养、互动的章节

```
[通用结构]

## 感情场景信息
- 感情阶段：[初识/神秘观察/开始试探/建立信任/好感萌生/暧昧/确认关系]
- 当前关系温度：[0-100分]（从角色状态卡获取）
- 场景目的：[建立信任/消除误会/感情升温/确认关系]

## 林婉儿特殊设定（写此类场景必须注意）
- 林婉儿是穿越者，有秘密，第四卷才暴露
- 前期她对主角有独特关注，但主动保持距离
- 不能让林婉儿行为"过于正常"，要体现她的神秘感
- 不要透露她是穿越者（第四卷之前）

## 互动设计
- 互动方式：[对话/共同行动/危机互助/日常相处]
- 感情推进要点：
  1. [如"两人因某事产生分歧"]
  2. [如"对话中暗含关心"]
  3. [如"一个细节体现默契"]

## 角色说话风格参考
【李长风典型对话】
"好。"（简短回答，不多废话）
"你知道什么？"（追问，不放过细节）
"跟紧我。"（行动时的简洁指令）

【林婉儿典型对话】
"小心玄苦。"（说完就走，留悬念）
"还能走吗？"（关心但不直说）
"我听过这种钟声。"（说半句留钩子）

## 注意事项
- 不要直白写"我爱你""我喜欢你"等，用行为和细节体现感情
- 感情推进要自然，不要突然升温（第一卷关系温度不超过40/100）
- 通过小动作体现关心（递药、挡路、默默跟随等）
- 林婉儿留悬念时要点到为止，不能说太多
```

- 角色关系：[如"李长风与张云义义兄弟互动"]
- 互动方式：[切磋武艺/聊天/共同完成任务/赶路聊天/整理物资]

## 对话设计
- 话题一：[如"讨论刚才战斗的得失"]
- 话题二：[如"聊起各自的过往/来历"]
- 话题三：[如"谈到下一步计划"]

## 李长风+张云义 互动特色参考
- 张云义喜欢说废话，李长风一句话截断
- 张云义激动，李长风冷静，互补性格

- 两人之间有默契：打仗前不需要太多说明

## 注意事项
- 对话要自然，不要像在接受采访
- 适当加入一些轻松/幽默元素（张云义性格适合这个）
- 日常互动中也要有小信息点推进，不能纯灌水
- 至少有一个细节体现兄弟之间的默契/信任
```

## 6. 游戏系统面板展示Prompt

> 适用于需要展示系统面板的场景（突破、获得奖励、任务完成等）
> 注意：系统面板只在关键时刻出现，不要滥用

```
[通用结构]

## 面板展示信息
- 触发原因：[为什么弹出面板？击杀BOSS/突破境界/完成任务/系统奖励]
- 展示时机：[战斗结束后/突破瞬间/任务完成后]
- 面板变化：[与上一次相比，哪些数值变化了]

## 突破场景设计（如本章有突破）
三步走法则（写突破时必须遵守）：
1. **内因积累**：战斗/修炼过程中，剑道领悟持续加深，剑意凝练（写感受）
2. **能量外因**：断剑吸收某种高纯度能量（写明颜色），或服用灵药，真气充盈
3. **系统确认**：【叮！大境界壁垒已破！XXX境界解锁！】（系统提示格式）

## 系统提示格式参考
```
【叮！[事件描述]！[获得内容]！】
```

示例：
```
【叮！击杀BOSS！获得经验值+50！自由属性点+1！】
【叮！剑道顿悟+真气溢出，突破条件满足！大境界壁垒已破，剑宗初期解锁！】
【叮！检测到灵根检测法器！启动被动技能·数据伪装：灵根数据已修改为顶级！】
```

## 注意事项
- 系统面板不要频繁出现，一章最多出现一次（有意义时才出现）
- 不要用系统面板堆数值，重点描写感受和成长
- 突破场景要有情感冲击力，不只是数字变化
- 系统乱码（F001a相关）是伏笔，如果出现要有意义
```

## 9. 转场/过渡场景Prompt

> 适用于章节之间的过渡、时间跳跃、地点转换

```
[通用结构]

## 转场信息
- 转场类型：[时间跳跃/地点转换/视角切换]
- 时间跨度：[多少天/多少时辰]（用故事内时间，不用章节数）
- 地点变化：[从哪里到哪里]

## 转场处理
- 转场写法：选一种
  1. 环境变化法：通过环境描写暗示时间/地点变化
  2. 角色动作法：通过角色的一个动作/一句话过渡
  3. 时间标注法：直接用"三日后""三个时辰后"等标注
- 转场中包含的信息点：[至少一个信息点：修炼进度/情报获得/关系变化]

## 转场后
- 转场后快速进入核心事件
- 转场后的第一个场景必须有足够吸引力

## 注意事项
- 转场必须自然，不要突然跳跃
- 转场中至少包含一个信息点
- 转场后快速进入核心事件
- 转场不要超过400字
- 不要出现"第X章"等元叙事
```
