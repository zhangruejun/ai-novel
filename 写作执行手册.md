# 写作执行手册（必读！）

> **⚠️ 这是最重要的文件。每次开始写作前，必须完整阅读本手册。**
> **⚠️ 如果你是新对话实例，请先读完本手册再执行任何操作。**

---

## 🚨 铁律：违反任何一条等于任务失败

1. **必须先读本文档，再读工作流.md，最后才开始写作**
2. **每章必须分2-4个场景生成，每个场景有独立的Prompt和草稿文件**
3. **写完每章后，必须更新角色状态卡、伏笔追踪表、每日写作日志**
4. **必须先做细纲→再做Prompt→再分场景生成→再拼接→再审校6a→再校验6b→最后定稿**
5. **不允许跳过任何步骤，包括步骤6b（审校校验）**
6. **步骤6b必须独立执行，禁止读取6a的审校报告**

## 每章写作8步流程（缺一不可）

> ⚠️ **v1.1更新**：步骤6拆分为6a（深度审校）+ 6b（审校校验），形成双审校机制，防止走过场。

### 步骤1：准备工作（15分钟）
- [ ] 打开 `references/角色状态卡.md`，确认当前角色状态
- [ ] 打开 `references/伏笔追踪表.md`，确认伏笔状态
- [ ] 打开 `references/时间线管理.md`，确认当前时间线和季节
- [ ] 打开 `volumes/volume01_穿越觉醒.md`,确然当前章节详细规划
- [ ] 打开 `写作执行手册.md`（就是本文档），回顾8步流程
- [ ] 回顾前3章定稿内容（快速浏览 `chapters/` 目录下最近3章）


### 步骤2：制作章级细纲（20分钟）
- [ ] 复制 `templates/章级细纲模板.md` 到 `outlines/volXX/chXXX_细纲.md`
- [ ] 填入本章核心事件
- [ ] 拆分2-4个场景，明确每个场景的核心动作
- [ ] 为每个场景指定**场景类型（A-F）**（见下方场景类型说明）
- [ ] 设定结尾钩子
- [ ] 填写细节要求
- [ ] 确认场景类型不连续3章相同组合

### 步骤3：生成Prompt（15分钟/场景）
- [ ] 从 `templates/Prompt模板集合.md` 选择合适的模板
- [ ] 为**每个场景**创建独立的Prompt文件：`prompts/volXX/chXXX_场景X.md`
- [ ] 填入细纲内容
- [ ] 附带角色状态卡（复制references中的相关内容）
- [ ] 附带禁用语列表（复制references/禁用语列表.md内容）
- [ ] 附带上一章结尾200-300字作为前情提要
- [ ] 明确范围限制（只写到什么为止，不要写什么）

### 步骤4：AI分场景生成（10分钟/场景）
- [ ] 将**每个场景**的Prompt发送给AI
- [ ] 等待AI生成（每场景800-1200字）
- [ ] 将生成内容保存到 `drafts/volXX/chXXX_场景X_草稿.md`
- [ ] **每个场景单独保存为独立文件**
- [ ] 快速检查是否符合场景要求
- [ ] 检查是否超出Prompt范围限制

### 步骤5：拼接与初校（20分钟）
- [ ] 将2-4个场景草稿**拼接成一章**
- [ ] 使用 `***` 作为场景分隔符
- [ ] 润色场景过渡段落，确保衔接自然
- [ ] 检查字数是否在2800-3200字范围内
- [ ] 初步检查是否有明显AI痕迹
- [ ] 保存为 `drafts/volXX/chXXX_完整草稿.md`

### 步骤6a：深度审校（30-40分钟）
- [ ] 复制检查清单到 `reviews/chXXX_检查清单.md`
- [ ] 逐项检查（角色一致性、剧情一致性、**时间线一致性**、力量体系、语言质量、节奏）
- [ ] 修改发现的问题
- [ ] 去除AI痕迹（替换禁用词汇、润色心理描写等）
- [ ] 注入个人风格（调整句式、增加细节）
- [ ] 填写**修改记录表**（见下方格式）
- [ ] 给出**审校评分**（见下方评分标准），必须≥80分

### 步骤6b：审校校验（独立子任务，15-20分钟）、
 **隔离规则**：步骤6b**禁止读取** `reviews/chXXX_检查清单.md`（6a的审校报告），必须独立判断。

- [ ] 对比审校定稿和完整草稿，**随机抽查5处修改**，确认修改真实有效
- [ ] 使用禁用语列表**全文扫描**审校定稿，确认无禁用词残留
- [ ] 给出**独立评分**，与6a评分对比，确认是否虚高
- [ ] 随机抽查3项一致性内容，独立验证
- [ ] 填写**校验报告**（见下方格式），保存为 `reviews/chXXX_校验报告.md`
- [ ] 如校验不通过，退回步骤6a重新审校

### 步骤7：保存定稿并更新各个状态卡（5分钟）
- [ ] 将定稿保存到 `chapters/第XXX章_章名.md`
- [ ] **更新 `references/角色状态卡.md`**
- [ ] **更新 `references/伏笔追踪表.md`**
- [ ] **填写 `logs/每日写作日志.md`**

## 场景类型说明（A-F）

在步骤2制作细纲时，必须为每个场景指定类型：

| 类型 | 名称 | 特点 | 适用场景 |
|------|------|------|----------|
| **A** | 对话主导 | 以对话推进剧情，角色交锋 | 谈判、交流、情报获取 |
| **B** | 动作主导 | 以行动为核心，战斗、追逐、探索 | 战斗、逃亡、探险 |
| **C** | 悬疑推理 | 发现线索、解密、推理过程 | 线索发现、案件调查 |
| **D** | 感情培养 | 感情互动、关系发展、信任建立 | 日常相处、危机互助 |
| **E** | 环境氛围 | 环境渲染、心理冲击、氛围营造 | 突破、觉醒、异变 |
| **F** | 转折冲击 | 突发事件、反转、危机降临 | 意外发生、真相揭露 |

### 场景拆分规则
1. 每章必须拆分**2-4个场景**（推荐3个）
2. 每个场景**800-1200字**
3. **场景类型不要连续3章相同组合**（轮换使用，避免模板化）
4. 场景之间必须有明确的**过渡方式**（环境变化/角色动作/时间流逝/角色思绪）
5. 每个场景必须有**明确的结尾**（自然过渡到下一场景或钩子）

## 审校评分标准（步骤6a使用）

| 评分项 | 满分 | 说明 |
|--------|------|------|
| 剧情推进 | 15 | 是否有明确的核心事件，推进是否有力 |
| 角色塑造 | 15 | 角色性格是否鲜明，行为是否符合设定 |
| 对话质量 | 15 | 对话是否自然，是否符合角色性格 |
| 战斗描写 | 10 | 战斗是否有策略性，是否有细节（如无战斗标N/A） |
| 情感表达 | 10 | 情绪层次是否分明，是否真实 |
| 节奏控制 | 10 | 节奏是否紧凑，是否有张有弛 |
| 结尾钩子 | 10 | 钩子是否有吸引力，是否驱动读者继续阅读 |
| 语言质量 | 10 | 整体是否流畅，AI痕迹是否轻微 |
| 一致性 | 5 | 是否符合设定，是否有矛盾 |
| **总分** | **100** | **≥80分通过，<80分需修改后重新审校** |

## 一致性检查清单（步骤6a逐项检查）

### 角色一致性
- [ ] 角色名字是否正确
- [ ] 角色当前境界是否正确
- [ ] 角色当前状态是否正确（是否有伤、是否在附近等）
- [ ] 角色性格表现是否符合设定
- [ ] 角色称呼是否一致
- [ ] 角色的说话风格是否符合其性格/身份/背景
- [ ] 角色的行为是否符合其当前动机
- [ ] 角色是否"消失又突然出现"

### 剧情一致性
- [ ] 时间线是否正确（是否跳跃/倒流）
- [ ] 地点是否正确（角色是否在合理的位置）
- [ ] 前文提到的物品/信息是否被正确使用
- [ ] 伏笔是否被正确处理（埋设/回收）
- [ ] 反派行动是否符合其动机
- [ ] 剧情推进是否合理
- [ ] 本章内容是否与大纲一致
- [ ] 与上一章的衔接是否自然流畅

### 时间线一致性 ⚠️ **新增**
- [ ] 当前章节的"Day X"是否与时间线管理文档一致
- [ ] 角色的年龄是否与时间线推算一致
- [ ] 季节/天气是否与时间线设定的季节对应
- [ ] 路程耗时是否合理（参考时间线管理中的路程耗时表）
- [ ] 角色修炼/闭关时间是否与外界流逝时间一致
- [ ] 任务时限是否按时计算（如新手任务截止日）

### 力量体系一致性
- [ ] 境界等级是否正确
- [ ] 战斗结果是否合理
- [ ] 技能使用是否符合设定
- [ ] 角色的战斗力表现是否符合其当前境界

### 语言质量
- [ ] 是否有禁用AI词汇
- [ ] 是否有现代用语
- [ ] 对话是否符合角色性格
- [ ] 是否有重复的句式或描写
- [ ] 是否有过度空洞的心理描写
- [ ] 是否有"注水"内容

## 常见问题处理

| 问题 | 解决方案 |
|------|---------|
| **字数不够**（少于2800字） | 回溯到步骤4，找到字数最少的场景，检查Prompt范围限制是否过严，要求AI扩展细节描写/对话内容/内心活动 |
| **审校不通过**（评分<80分） | 查看审校报告，分类问题类型（语言质量/角色一致性/剧情一致性），逐项修改后重新评分。连续2章<75分触发质量预警 |
| **场景衔接生硬** | 检查细纲过渡方式是否合理；使用环境变化/角色动作/时间流逝/角色思绪过渡；场景结尾留引子，下一场景开头承接 |
| **AI忘记前文设定** | 在Prompt前情提要中明确列出关键信息（谁在场、持有什么、受伤情况等）；增加信息密度，不要只写在角色状态卡里 |
| **AI过度发挥** | 在Prompt范围限制中明确列出禁止事项（只写到X为止、不要写Y、不要添加大纲外的角色）；发现后删除超范围内容重新生成 |
| **模板化内容** | 检查场景类型轮换（不连续3章相同组合）；尝试不同写法（视角切换/时间跳跃/倒叙插叙）；启动质量预警处理 |

## 常见错误及后果

| 错误 | 后果 | 如何避免 |
|------|------|---------|
| 不分场景，一章一个Prompt | 质量差，无法控制细节 | 每章必须2-4个场景，每个场景独立文件 |
| **不更新角色状态卡** | 后续章节角色状态混乱 | 每章完成后立即更新 |
| **不更新伏笔追踪表** | 伏笔遗漏或重复 | 每章完成后立即更新 |
| **不写每日写作日志** | 进度无法追踪 | 每章完成后立即填写 |
| 跳过细纲直接写 | 结构混乱，偏离大纲 | 必须先做细纲 |
| 不分场景直接生成 | 质量差，AI容易跑题 | 必须分场景生成 |
| **跳过步骤6b（审校校验）** | 审校走过场，质量问题无法发现 | 6b必须作为独立子任务执行，禁止读取6a报告 |
| **不指定场景类型** | 连续章节场景类型重复，读者感觉模板化 | 每章细纲必须指定A-F类型，不连续3章相同组合 |
| **不检查时间线一致性** | 时间跳跃/倒流、季节错误、路程耗时不合理 | 步骤6a逐项检查时间线一致性清单 |
